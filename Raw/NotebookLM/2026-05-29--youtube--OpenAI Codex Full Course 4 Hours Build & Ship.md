### 一、Codex 核心定义：本地赋能的智能编码工具

Codex 是 OpenAI 推出的**本地运行型智能编码工具**，与网页版 ChatGPT/Claude 核心差异在于「深度本地交互能力」：

- **核心突破**：无需 sandbox 环境，直接访问本地文件系统、运行终端命令、操控桌面应用，实现「从建议到执行」的闭环
- **类比理解**：网页版 ChatGPT 像「给你提建议的朋友」，而 Codex 像「坐在你电脑前帮你干活的助手」
- **适用场景**：
    - 自动化任务：邮件起草、PDF 批量分析、Excel 数据统计、税务处理
    - 开发场景：本地代码编写、多文件协同、应用调试与部署
    - 跨工具联动：控制浏览器、Office 套件、设计软件等第三方应用

### 四、上下文窗口与压缩：额度管理关键技巧

#### 1. 上下文窗口核心规则

- 额度消耗：文件内容、对话历史、工具输出（如终端日志）均占用 token 额度
- 限制场景：长文件分析、多代理并行任务时易触发额度上限
- 优化方案：**Compaction（压缩）功能**
    - 自动压缩：系统定期汇总对话关键信息，删除冗余内容释放额度
    - 手动触发：输入框输入 `/compact` 立即压缩当前线程
    - 效果：保持核心上下文的同时，减少 60%-80% 的 token 占用

### 五、语音输入技巧：提升指令效率

讲师强烈推荐语音输入，核心优势与使用要点：

- **效率优势**：语速 150 词 / 分钟（远超打字速度），适合传递多步骤、复杂逻辑的指令
- **质量提升**：语音指令更完整，减少打字遗漏，模型理解准确率提升 30%+
- **工具支持**：
    - 内置语音：直接点击麦克风图标，支持实时转写
    - 第三方工具：推荐 Wispr Flow，支持方言识别、指令修正

### 六、权限与安全：平衡效率与风险

#### 1. 权限分级策略

表格

|权限级别|适用场景|安全保障措施|
|---|---|---|
|默认权限|日常办公、轻度开发|限制文件删除、系统修改等高危操作|
|Full Access|重度开发、信任场景|开放全部本地权限，需手动启用|

#### 2. 安全工具推荐

- **Destructive Command Guard**：禁止 `rm -rf`、系统重置等高危命令
- **权限审计日志**：记录代理所有文件操作、终端命令，支持回溯排查
- **临时权限模式**：针对敏感任务，临时开放权限，任务结束自动回收

### 七、AGENTS.md 文件：谨慎使用的项目配置

#### 1. 核心作用

项目根目录（或子文件夹、全局目录）下的说明文件，Codex 启动时自动读取，用于补充项目上下文：

- 支持层级：全局（用户目录）> 项目 > 子文件夹
- 适用场景：仅当模型对项目认知持续偏差时使用（如指定特殊依赖版本、非标准目录结构）

#### 2. 讲师核心建议：不盲目创建

- 负面效果：
    - 冗余信息占用上下文额度
    - 项目结构变更后易产生冲突，误导模型决策
- 研究佐证：
    - 新模型（如 GPT-5.2+、Sonnet 4.5）：AGENTS.md 无明显增益，甚至降低效率
    - 旧模型（如 GPT-5.1 Mini）：人类编写的 AGENTS.md 可小幅提升准确率
- 正确用法：仅补充「模型无法自行推断」的静态规则（如 `使用 Tailwind v4 而非 v3`、`Python 版本必须为 3.14`）

### 四、技能与 MCP 区别：场景化选择核心（<start>01:48:24<end>）

### 五、子代理用法：并行任务的效率利器（>01:58:18）

#### 核心价值：拆分任务，并行执行

子代理（Sub-agents）是 Codex 实现「多线程工作」的核心工具，本质是**让主代理拆分复杂任务，创建多个子代理同步执行，大幅缩短总耗时**。

#### 适用场景

- 批量处理文档（如 10 篇 PDF 汇总、多份论文分析）；
- 多模块并行开发（如前端组件开发 + 后端接口编写同步进行）；
- 重复性任务批量执行（如多文件格式转换、多平台内容发布）。

#### 操作示例（PDF 汇总任务）

1. 下达指令：
    
    plaintext
    
    ```plaintext
    1. 进入「AI 就业影响」文件夹下的「研究论文」目录；
    2. 创建 10 个子代理，每个子代理负责读取1篇 PDF；
    3. 每个子代理生成包含「核心观点+YouTube 选题角度」的摘要（Markdown 格式）；

    ```
    
2. 关键特性：
    - 自动拆分任务，无需手动分配；
    - 并行执行，10 篇 PDF 汇总耗时 = 单篇处理耗时（而非累加）；
    - 主代理负责结果整合与异常处理（如某篇 PDF 读取失败时重试）。

### 六、工具选择原则：从简单到复杂的决策逻辑（>02:08:47）

#### 核心原则：「最小成本优先」，按需求复杂度升级

1. **新手入门 / 通用需求**：优先用「插件」

2. **个性化流程 / 复杂需求**：用「技能」
    - 插件无法满足的定制化场景（如「转录稿→教程」转换、支持工单自动化）；
3. **通用外部服务对接**：用「MCP」
    - 需调用自定义 API 或小众服务（如企业内部系统、私有数据库）；
4. **批量并行任务**：用「子代理」
    - 多文档处理、多模块开发等需要同步执行的场景；
5. **桌面软件操作**：用「电脑控制」
    - 无 API 接口的软件（如本地设计工具、客户端应用）、自动化测试等场景。

plaintext

```plaintext

                                  → 否 → 需桌面操作？→ 是 → 电脑控制
                                                    → 否 → 需对接外部系统？→ 是 → MCP+技能
                                                                          → 否 → 纯流程自动化 → 技能
```

### 一、Git/GitHub 基础（02:17:00）

#### 核心定位

Git 是分布式版本控制系统，核心价值是**追踪文件变更、回溯历史版本、支持多人协作**；GitHub 是 Git 仓库的云端托管平台，Codex 深度集成二者，让非技术人员也能零代码操作。

#### Codex 自动化能力

1. **自动版本管理**：无需手动输入 `git commit`/`git push`，在 Codex 桌面端直接通过可视化界面完成提交（填写提交信息、选择变更文件）、分支创建 / 切换、合并请求（PR）发起。

3. **协作简化**：多人协作时，Codex 可自动拉取远端分支更新、解决简单冲突（如文本文件冲突），并生成协作日志，让非技术人员也能参与团队开发。

#### 实操场景

- 新建项目后，Codex 可自动初始化 Git 仓库并完成首次提交；

### 七、工作树并行开发（03:23:55）

#### 核心概念

工作树（Worktrees）是 Git 的进阶功能，允许在**同一仓库下创建多个独立工作目录**，实现多分支并行开发，互不干扰（类似「多副本开发」但共享 Git 历史）。

#### Codex 强化优势

1. **可视化管理**：无需手动输入 `git worktree` 命令，在 Codex 桌面端直接创建、切换、删除工作树；
2. **并行任务处理**：如同时开发「首页优化」和「功能模块」两个需求，可创建两个工作树，分别对应不同分支，Codex 可在多个工作树中同时执行任务（如一边写代码、一边测试）；
3. **协作高效**：团队成员可通过工作树共享分支，Codex 自动同步各工作树的变更，避免分支冲突。

#### 适用场景

- 多功能并行开发（如同时开发 2-3 个独立需求）；
- 紧急 Bug 修复（在不影响主分支开发的前提下，创建临时工作树快速修复）。

通过统一配置文件确保所有工作树的「环境一致性」，避免依赖冲突或密钥泄露，核心配置文件：

1. **environment.toml**：存储全局环境变量（如 API 密钥、数据库连接地址、依赖版本号），所有工作树共享此文件，Codex 自动读取；
2. **启动脚本（start.sh/start.bat）**：统一配置依赖安装、服务启动命令（如 `npm install && npm run dev`），Codex 可一键执行，确保所有工作树启动流程一致。

#### 配置步骤（Codex 自动化辅助）

1. 在仓库根目录创建 `environment.toml`，Codex 可自动生成模板并提示填写必要参数；
2. 编写启动脚本，Codex 可根据技术栈自动生成命令（如 Next.js 项目的启动命令）；
3. 新建工作树时，Codex 自动复制配置文件，无需手动同步。

### 十、Codex 驱动迭代（03:44:57）

#### 迭代核心逻辑

Codex 实现「全流程自动化迭代」，从 Bug 修复到功能优化，无需人工干预核心流程：

1. **Bug 自动修复**：通过自动化巡检或用户反馈的 Issues，Codex 定位代码问题，自动生成修复代码并提交 PR；
2. **功能优化**：输入自然语言指令（如「优化轮播图切换动画，增加淡入效果」），Codex 自动修改前端代码，生成优化 PR；
3. **PR 自动生成**：完成开发 / 修复后，Codex 自动创建 PR，填写变更说明、关联 Issues，甚至生成测试用例；
4. **持续监控**：部署后，Codex 可监控应用运行状态（如报错日志），自动创建 Bug Issues 并触发修复流程。

#### 迭代优势

- 效率提升：人工仅需确认需求和 PR 结果，90% 开发工作由 Codex 完成；
- 低出错率：基于 Git 版本控制和自动化测试，减少人工操作失误。
