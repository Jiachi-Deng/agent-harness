# 二、前置环境：Node.js 安装与校验（必装依赖）

Codex CLI（命令行版）**强制依赖 Node.js 运行环境**，这是安装失败的头号原因。

## 2.1 环境校验（判断是否已安装）

全平台通用操作（Mac / Windows / Linux）：

1. 打开系统终端：
    
    1. Mac：聚焦搜索输入 `终端` 打开；
        
    2. Windows：按下 `Win + R`，输入 `cmd` 打开命令提示符；
        
2. 输入校验命令：
    
    ```bash
    node -v
    ```
    
3. 结果判断：
    
    1. 输出版本号 → 环境正常，可直接下一步；
        
    2. 无输出 / 报错 → 未安装 Node.js，必须下载安装。
        

## 2.2 Node.js 安装步骤

1. 打开视频附的官方下载地址，选择**最新稳定版**；
    
2. 根据你的操作系统（Mac/Windows/Linux）选择对应安装包；
    
3. 按照页面给出的终端命令逐行执行，全程自动化安装；
    
4. **关键注意**：安装完成后**重启终端**，否则系统无法识别新环境。

# 五、核心参数配置：模型、推理强度、高速模式

这三组参数直接决定 Codex 的**输出质量、运行速度、额度消耗**，是使用的核心。

## 5.1 AI 模型选择（硬性规则）

1. 核心原则：**永远选择 OpenAI 当下最新的 GPT 正式模型**（视频录制时为 GPT 5.4，后续会迭代 GPT 5.5 等新版本）；
    
2. 切换模型指令：CLI 中输入 `/model`，调出模型列表直接选择；
    
3. ❌ 绝对不要使用的模型（避坑重点）：
    
    1. 旧版本：GPT 5.2、GPT 5.3 Spark、GPT 5.3 Codex；
        
    2. 轻量阉割版：所有 Mini 模型（代码逻辑、排错能力严重不足，无法做项目开发）。
        

## 5.2 推理强度（Reasoning Effort）共 4 档

控制 AI 分析代码、思考问题的深度，默认档位为 `Medium（中等）`，各档位场景划分：

1. **Low（低推理）** 完全不推荐使用。思考深度不足，代码漏洞、逻辑错误频发，输出质量差。
    
2. **Medium（中等，默认）** 适用：简单脚本、代码片段、日常小幅修改；想节约额度的入门用户。 特点：速度均衡，额度消耗适中。
    
3. **High（高推理，黄金档位）** 主讲人日常主力档位，**90% 项目开发首选**。 适用：完整项目开发、功能迭代、常规 Bug 修复。 特点：智能适配任务：简单修改 5 秒完成；复杂任务自动延长思考时间（2~4 分钟），兼顾速度与稳定性。
    
4. **Extra High（超高推理）** 仅用于**极端复杂场景**：深层疑难 Bug、全站代码重构、高风险后端改造。 特点：思考时间最长、代码最严谨，但额度消耗极快，非必要不开启。
    

## 5.3 /fast 高速模式

1. 开关指令：CLI 输入 `/fast`（切换式，再输一次关闭）；
    
2. 效果：代码生成、推理速度**直接翻倍**，长任务不再卡顿；
    
3. 适用人群划分：
    
    1. ✅ 推荐开启：$$100$$200 Pro 订阅用户（额度充足）；
        
    2. ❌ 不推荐：$$8$$20 入门 / Plus 用户（会快速用尽额度）。

# 六、终端基础命令 & 项目目录规范（零基础必学）

视频强调：哪怕完全不懂编程，只需掌握 4 条终端命令 + 目录规范，就能安全使用 Codex。

## 6.1 四条核心终端命令（全平台通用）

1. `pwd`：查看**当前文件夹的完整路径**，用于定位目录；
    
2. `ls`：列出当前文件夹内所有文件、子文件夹，查看目录内容；
    
3. `cd 文件夹名`：切换目录（进入指定文件夹）
    
    1. 快捷技巧：输入文件夹首字母后按 `Tab` 键，自动补全文件夹名；
        
    2. 返回上一级目录：`cd ..`（固定写法，两个英文点）；
        
4. `clear`：清空终端界面。
    

## 6.2 项目目录规范（编程核心习惯）

### 错误做法（强烈禁止）

在**系统根目录、电脑桌面**直接启动 Codex。 风险：文件杂乱、权限混乱，极易误删系统文件，属于业余操作。

### 标准规范（强制遵守）

1. **一个项目 = 一个独立文件夹**，所有项目分类存放；
    
2. 实操流程：
    
    1. `ls` 查看根目录文件夹 → `cd 文档` 进入总目录；
        
    2. 逐层 `cd` 进入你的**项目专属文件夹**；
        
    3. 在此目录下执行 `codex` 启动工具；
        
3. 优势：Codex 仅对当前文件夹生效，文件隔离、安全可控。

# 九、IDE 集成：Cursor / VS Code 搭配 Codex 开发

CLI 适合快速启动，但**复杂项目必须使用 IDE（集成开发环境）**，可视化管理文件、代码、终端。

## 9.1 IDE 基础概念

IDE = 集成开发环境，整合「代码编辑器 + 终端 + 调试工具 + 版本控制 + AI 助手」。

1. **VS Code**：微软开源免费，全球最流行 IDE，Cursor、Windsurf 等工具均基于它二次开发；
    
2. **Cursor**：主讲人主力 IDE，内置多模型 AI 助手，完全兼容 VS Code 插件，免费入门。
    

## 9.2 Cursor 安装与基础操作

1. 下载地址：`cursor.com`，匹配系统版本下载安装；
    
2. 核心快捷键（通用）：
    
    1. `Ctrl + B` / `Command + B`：显示 / 隐藏文件侧边栏；
        
    2. `Ctrl + J` / `Command + J`：打开 / 关闭**集成终端**（核心功能）；
        
    3. `Ctrl + I` / `Command + I`：显示 / 隐藏 AI 助手侧边栏；
        
3. 集成终端优势：打开后**自动定位到当前项目文件夹**，无需手动切换目录，直接启动 Codex。
    

## 9.3 YOLO 全权限模式（codex --yolo）

1. 启动指令：
    
    ```bash
    codex --yolo
    ```
    
2. 功能：跳过所有权限确认弹窗，赋予 Codex **完整读写、运行权限**，对标 Claude Code 的免权限命令；
    
3. 适用：本地可信项目、快速开发，减少弹窗干扰；
    
4. ⚠️ 安全提醒：**仅在自己信任的本地项目使用**，陌生文件、外网项目禁止开启。
    

## 9.4 Codex IDE 扩展插件（VS Code / Cursor 通用）

1. 安装流程：左侧「扩展」→ 搜索 `OpenAI Codex`（官方插件，300 万 + 安装量）→ 一键安装；
    
2. 两种形态对比：

# 十、API 对接、调试技巧 & 提示词工程

## 10.1 Open Router API 对接（项目核心第三方服务）

Open Router 是 AI 模型聚合平台：**一个 API 密钥，调用全球所有主流模型（GPT/Gemini/Claude）**。

### 账号 & 密钥创建流程

1. 访问 `openrouter.ai` 注册账号（免费入门，充值 5~10 美元足够测试）；
    
2. 右上角进入 `Settings → API Keys`；
    
3. 点击 `Create` 创建密钥，命名项目名称，**强制设置额度上限**（防止超额扣费）；
    
4. 复制密钥，妥善保管，禁止公开。
    

### 项目配置

1. 在项目根目录创建 `.env` 环境变量文件，存放 API 密钥；
    
2. 规范优化：新建 `/docs` 文件夹，创建 `open-router-nano-banana-syntax.md`，存入官方 API 文档、图片传参规则，方便 AI 查阅。
    

## 10.2 标准报错调试流程（实战图像生成失败案例）

1. 截图报错界面，粘贴给 Codex，直观展示问题；
    
2. 前端排查：按 `F12` / `Command + Option + I` 打开浏览器控制台，查看前端日志；
    
3. 服务端排查：查看终端运行日志，定位后端错误；
    
4. 深度调研：用 Perplexity 检索官方调用规范，补充进项目文档；
    
5. 代码优化：将**串行 API 调用改为并行独立调用**，解决 “一个任务失败、全部卡住” 的问题。
    

## 10.3 Prompt 提示词工程（核心方法论）

视频用行业案例总结三类使用者，给出最优写法：

1. 新手（低阶）：提示词过短、描述模糊 → AI 理解偏差；
    
2. 过度钻研者（中阶）：堆砌超长模板、无关信息过多 → 效率极低；
    
3. 高手（高阶）：**简洁 + 精准**，只写核心需求，边开发边迭代。
    

### 通用原则

- 拒绝模糊，明确功能、约束、格式；
    
- 拒绝冗余，不堆砌无效模板；
    
- 善用图片、文档作为上下文补充；
    
- 一次只下达一组需求，循序渐进。
    

## 10.4 Git 版本控制（代码必备工具）

Git 用于记录代码修改、版本回滚，**哪怕不上传云端，本地项目也必须使用**。

1. 基础操作（全部交给 Codex 自动执行）：
    
    1. 初始化仓库：`在当前项目初始化 Git 仓库`；
        
    2. 创建 `.gitignore`：自动配置忽略文件（密钥、依赖包、日志等敏感文件）；
        
    3. 提交代码：`暂存所有修改并提交，自动生成提交说明`；
        
2. 最佳实践：**每 10~15 分钟提交一次代码**，频繁保存版本，防止代码丢失。

# 十二、高阶功能：沙箱、桌面端、插件、自动化、子代理

## 12.1 沙箱模式 & 审批策略（安全防护）

Codex 内置安全沙箱，限制高危操作，防止误删文件、执行恶意脚本：

1. 三种沙箱模式：划分 Codex 可操作的文件 / 系统范围；
    
2. 四种审批策略：控制删除、改配置、运行脚本等高危操作是否需要人工确认；
    
    1. 新手 / 陌生项目：开启严格审批；
        
    2. 本地可信项目：搭配 YOLO 免确认模式。
        

## 12.2 Codex 桌面客户端

独立桌面端，区别于 CLI 和 IDE 插件：

1. 核心能力：多代理并行管理、会话分类归档、多窗口分栏；
    
2. 适用：重度用户、同时维护多个项目的开发者。
    

## 12.3 技能插件（Skills）

类似浏览器插件，扩展 Codex 能力：

1. 安装：CLI / 客户端内在线安装官方 / 社区插件；
    
2. 常用插件：图像生成、代码测试、接口调试、文档生成；
    
3. 用法：安装后直接在提示词中调用，无需额外配置。
    

## 12.4 自动化任务

替代传统代码脚本，用自然语言配置定时任务：

- 适用：批量文件处理、定时代码检查、日志整理；
    
- 优势：零基础也能搭建自动化流程，不用学习脚本语言。
    

## 12.5 子代理（Sub Agents）

大型项目多任务调度核心功能：

1. 原理：主代理负责核心逻辑，把简单重复任务拆分给**多个子代理并行处理**；
    
2. 优势：主会话不阻塞，大型项目拆解更清晰，整体效率翻倍。

# 十三、全流程避坑总结（新手必看）

1. 环境：安装 Node.js 后**务必重启终端**，否则识别失败；
    
2. 模型：只选用最新正式 GPT 模型，拒绝旧版、Mini 轻量模型；
    
3. 推理档位：日常开发用 High，排错 / 重构用 Extra High，禁用 Low；
    
4. 额度：`/fast` 高速模式仅 Pro 订阅开启，入门版慎用；
    
5. 文件管理：禁止在桌面 / 根目录启动 Codex，项目分文件夹存放；
    
6. 安全：API 密钥、`.env` 文件绝不外泄，陌生项目不开 `--yolo`；
    
7. 提示词：简洁精准为主，不写超长冗余内容；
    
8. 版本控制：所有项目必用 Git，高频提交代码；
    
9. 调试：善用截图、官方文档、并行调用解决接口报错。
