# AI一人营销团队

## 解决什么问题

一个创作者、独立开发者或小团队，通常没有完整的内容团队、设计团队、视频团队、商务团队和社媒运营团队。AI 一人营销团队的目标不是让 AI “全自动做营销”，而是把营销拆成一组可重复调用的小工作流：

- 研究：从 YouTube、个人知识库、历史内容里找素材和模式。
- 选题：把保存过的材料变成可执行的内容想法。
- 脚本：生成开头、脚本、大纲、解释版本。
- 视觉：生成图表、缩略图、品牌视觉、视频覆盖层。
- 媒体：生成图片、视频、B-roll、演示动画。
- 商务：过滤品牌合作、整理邮件、建议会议时间。
- 发布：把候选内容同步到社媒排期工具。

这套方法的核心不是某一个工具，而是“grounding + skills + mini apps + automations + human final judgment”。

## 适合谁

适合：

- 做个人品牌、短视频、YouTube、课程或独立产品营销的人。
- 想把 Codex、Claude Code、技能文件、插件、自动化组合起来的人。
- 小团队里同时负责内容、增长、商务和运营的人。

不适合：

- 完全没有内容方向，只想让 AI 代替定位的人。
- 需要强品牌安全、法务审核、多人审批的大公司流程。
- 不愿意维护 API key、数据库、素材库和自动化权限的人。

## 可复用工作流

### 1. 先给模型 grounding

不要直接让 AI “写一个爆款视频脚本”。先给它可学习的参考点：

- 目标 YouTube 创作者的最近视频字幕。
- 自己过去的视频、文章、推文。
- Readwise / 剪藏 / 笔记里的高频兴趣点。
- 具体行业、产品或受众背景。

没有 grounding 的内容生成，往往会变成通用鸡汤；有 grounding 后，AI 才能提取结构、语气、节奏和选题模式。

### 2. 把一次满意结果沉淀成 skill

先手工跑出一次满意结果，再把步骤固化成 skill。不要上来就追求自动化。

判断一个流程是否值得做成 skill：

- 输入是否稳定，比如视频 URL、保存时间段、目标创作者、产品截图。
- 输出是否稳定，比如 30 个选题、5 个开头、品牌合作优先级表。
- 是否每周或每天都会重复。
- 是否能通过修改 skill 逐步变好。

### 3. 把高频 skill 变成 automation

只有经过几次人工使用和修正后，才适合自动化。比如：

- 每天早上 8 点读取过去 3 天 Readwise 收藏，生成 30 个短视频选题。
- 每天扫描商务邮件，生成品牌合作优先级表。
- 每周汇总最近研究、聊天和素材，放入 Buffer idea queue。

自动化适合“整理、汇总、候选生成”，不适合直接替你发布、签约或承诺。

### 4. 用 mini app 承接人机协作

Gen Media 这类模块最有启发：它不是纯聊天，也不是纯工具，而是一个人和 agent 都能操作的本地 mini app。

好的 mini app 应该让 agent 做前 90%：

- 调 API。
- 批量生成候选。
- 写入素材库。
- 加入网格或草稿区。

再让人做最后 10%：

- 挑最好的版本。
- 改图、改字、改构图。
- 判断是否符合品牌和语境。

### 5. 把发布当成收纳系统，而不是自动发射器

Buffer Publisher 的启发是：分发模块第一步应该是 idea inbox，不是自动发布。agent 可以把最近研究、聊天、Readwise 选题和 YouTube 脚本候选同步到 Buffer / Notion / Obsidian，但默认只创建 idea 或草稿。

真正发布前，人仍然要检查：

- 这个选题是否还符合当前定位。
- 来源是否可回查。
- 语气是否像自己。
- 平台格式是否合适。
- 时机是否合适。

### 6. 高权限模块必须有人类审批点

Email Manager 这类模块可以很实用，但权限风险明显更高。它适合先做“读取、筛选、汇总、建议”，不适合上来就“自动回复、自动约会、自动承诺”。

默认边界应该是：AI 可以读、整理、起草；发送邮件、创建会议、承诺价格和交付范围都必须等人确认。

## 模块落地细节

### Gen Media

Gen Media 的核心不是“让 AI 生成一张图”，而是做一个人和 agent 都能使用的素材工作台。第一版不必复杂，但至少要有三件事：

- API 层：少量高频模型入口，例如图像生成、image-to-video、upscale。
- 素材库：保存人物照片、品牌元素、参考图、生成参数、prompt 和标签。
- Grid UI：让人能批量查看候选、挑选版本、微调文字、背景和构图。

推荐顺序是先做最小 app：输入任务，生成 4 个候选，写入 grid；再加入人物、品牌色、参考缩略图等可复用元素；最后再让 agent 批量调用同一套能力。

它的关键判断是：agent 做前 90%，人做最后 10%。如果生成物只留在聊天里，不能被挑选、编辑、追踪和复用，这个模块就没有真正成立。

适合的操作指令骨架：

```text
请使用 Gen Media mini app 生成 4 个缩略图或封面候选。

主题：
[主题]

人物 / 品牌元素：
[元素名称]

风格参考：
[参考缩略图 / 创作者 / 品牌方向]

要求：
1. 选择合适的图片或视频模型。
2. 生成结果写入素材 grid。
3. 每个候选保存 prompt、模型、参数和参考元素。
4. 不要直接定稿，等我在 app 里挑选和微调。
```

常见失败点是只做 API 调用、不做素材管理；不保存参数，导致优秀结果不可复现；参考素材版权和肖像授权不清晰；第一版接入太多模型，反而无法稳定使用。

### Buffer Publisher

Buffer Publisher 的第一价值不是自动发布，而是建立 `idea queue`。它负责把 agent chats、Readwise 选题、YouTube Researcher 脚本候选和临时灵感收进一个统一入口，避免好点子散落在聊天记录里。

最小版本只做三件事：

1. 从最近研究、聊天和知识库里抽取候选 idea。
2. 每条 idea 带标题、角度、来源链接、适合平台和下一步动作。
3. 写入 Buffer、Notion、Obsidian、Markdown 或其它内容看板。

先不要自动发布。稳定以后，可以生成平台草稿；真正发出之前仍然需要人检查语气、时机、格式、来源和品牌一致性。

适合的操作指令骨架：

```text
请查看最近的聊天、研究和内容选题，把最值得保留的 5 个 idea 加入内容看板。

每个 idea 包含：
1. 标题
2. 一句话角度
3. 来源链接或来源页面
4. 适合平台
5. 下一步动作

限制：
- 只创建 idea 或草稿。
- 不要自动发布。
- 不要删除已有内容。
```

这个模块适合早期复刻，因为权限风险低、输出容易人工检查，并且能连接前面的 YouTube Researcher 和 Readwise 工作流。主要风险是 idea queue 变成垃圾桶，所以必须有每日或每周清理机制。

### Email Manager

Email Manager 的价值是减少低价值阅读和筛选，不是替人做商务决策。它可以读取 Gmail 和 Calendar，把品牌合作、赞助、客户咨询和会议需求整理成优先级表。

推荐从只读摘要开始：

1. 只筛选和总结，不回复。
2. 加入品牌匹配、金额线索、受众匹配、重复邮件和低质模板过滤规则。
3. 生成高 / 中 / 低优先级表。
4. 读取 Calendar 只建议时间，不创建会议。
5. 人确认后，才允许起草回复。
6. 再次确认后，才发送邮件或创建会议。

适合的操作指令骨架：

```text
请作为我的商务邮件经理，只做筛选和建议，不要发送邮件。

范围：
过去 72 小时 Gmail 中与品牌合作、赞助、商务邀约有关的邮件。

请输出：
1. 高 / 中 / 低优先级表格。
2. 每家公司为什么适合或不适合我。
3. 需要我回复的建议。
4. 如果需要开会，请读取 Calendar 并给出下周可选时间。

限制：
- 不要自动回复。
- 不要自动创建会议。
- 所有外部动作都等我确认。
```

这个模块比内容选题和媒体素材更危险，因为它接触邮箱、日历、商务机会和真实对外承诺。默认边界是：AI 可以读、整理、建议和起草；不能未经确认发送邮件、创建或修改日程，也不能承诺价格、档期和交付范围。

## 常见坑

- 把“营销团队”理解成一个巨大的万能 agent，而不是一组小 skill。
- 没有保留原始来源链接，导致选题无法回查。
- 模仿创作者风格过度，变成抄袭或失去自己的声音。
- 过早自动发布，跳过人工判断。
- 只做聊天 prompt，没有做素材库、idea queue、审批点这类可持续的工作台。
- 让 AI 直接处理 Gmail、Calendar、社媒账号，但没有权限边界和审批点。
- 把创作者视频里的产品宣称当成已验证事实。工具能力、API、价格和可用性都需要后续单独验证。
