# AI知识库

用 AI 管理个人资料、收藏、笔记、高亮、剪藏和第二大脑，并把它们转化成选题、教程、产品判断和业务动作。

这个主题的重点不是“存更多资料”，而是让资料能被反复调用、回查、组合和产出。

## 页面角色

| 页面 | 角色 | 合并判断 |
| --- | --- | --- |
| [[个人知识库到内容选题|个人知识库到内容选题]] | 当前主干页，负责“收藏 / 剪藏 / Readwise -> 内容选题” | 继续吸收 Readwise、Reader、Clippings、Obsidian、个人高亮到内容生产的证据和 SOP |

## AI 能召回什么 / Wiki 应沉淀什么

| 类型 | 处理方式 |
| --- | --- |
| AI 已经很熟的通用概念，如第二大脑、Zettelkasten、标签、摘要、检索、RAG 基础区别 | 不写百科长文，主页面保留问题框架、讲解路径和核验点；需要交付时再生成知识召回 prompt Output |
| 个人收藏如何变成真实选题、哪些来源长期有用、输出格式怎么稳定、自动化哪里会制造噪音 | 进入 Wiki 主页面，作为本库的经验、判断和 SOP |
| Readwise / Obsidian / Notion / Reader 的最新 UI、导出限制、API 变化、价格和同步问题 | 只有经过实测或可靠材料后再写入，并标注日期 |

## 合并原则

- 新材料如果讲的是“从个人资料长出选题”，优先并入 `个人知识库到内容选题`。
- 不要把本主题全部并入 `AI一人营销团队`；后者是编排页，本主题是可复用的 grounding 方法。
- 只有当材料自然膨胀出稳定分支时，才拆出新页，比如“收藏系统搭建”“资料回查与引用”“内容选题自动化”“知识库腐烂治理”。
