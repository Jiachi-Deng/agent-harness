# 讲解图与视觉画布工作流

## 解决什么问题

很多 AI 生成的视频脚本只有文字，没有视觉结构。讲解图和视觉画布的价值，是把概念、脚本、课程结构或产品卖点变成“视频里能看懂、能指着讲、能继续修改”的视觉材料。

这里不把 Excalidraw、Paper 这类工具拆成独立工具目录，而是按输出任务维护：

- Excalidraw：更适合低保真、低文本密度、能全屏讲解的视觉骨架。
- Paper：更适合高保真、可编辑、可截图反馈、可导出或转网页的视觉稿。

工具本身会变，长期稳定的是：先把脚本结构压成视觉关系，再让人通过画布检查、删减、重排和定稿。

本页负责“视频前期和中期的视觉结构”，不是剪辑时间轴。只要问题是概念怎么画、画布怎么改、缩略图或 landing page 视觉方向怎么定，就进入本页；如果问题变成第几秒出现、怎么动、是否遮挡、如何 render 成 overlay 或 MP4，就进入 [[Wiki/AI视频制作/AI视频剪辑与动态图形工作流|AI视频剪辑与动态图形工作流]]。

## 适合的任务

- 把一个复杂概念变成 3-5 张讲解图。
- 为视频或课程生成手绘风大纲。
- 把 YouTube transcript、个人知识库和当前主题合并成一套图。
- 生成适合全屏讲解的 mind map / flow。
- 为 PPT 或课程讲义准备初版视觉结构。
- 生成 landing page、缩略图、品牌视觉或 lead magnet 的初版画布。
- 把可视化画布交给 Codex / Claude Code 继续生成网页或视频素材。

## 输入

- 主题：要解释什么概念。
- 目标观众：新手、创业者、开发者、课程学员等。
- 风格参考：自己的视频、目标创作者 transcript、过往图表。
- 知识来源：Readwise 收藏、clippings、YouTube transcript。
- 图表偏好：文字少、节点少、层级清晰、可全屏讲解。
- 导出要求：Excalidraw 链接、PNG、SVG、网页原型、视频素材等。

## 推荐流程

1. 先让 AI 用文字列出讲解结构，不要直接画。
2. 让 AI 把结构压缩成 3-5 张图或 1 个可编辑画布，每张图只讲一个关系。
3. 根据输出目的选择画布：低保真讲解用 Excalidraw，高保真视觉原型用 Paper。
4. 如果需要同时查 YouTube 和 Readwise，使用 sub agents 并行做资料提取。
5. 打开画布，人工全屏检查或截图反馈。
6. 删除多余文字，调整节点位置、层级和视觉重心，把图改成自己能讲、能发布或能转网页的样子。
7. 把最终图用于视频、课程、PPT、图文卡片、缩略图、landing page 或网页原型。

## 什么时候用 Excalidraw，什么时候用 Paper

| 维度 | Excalidraw | Paper |
| --- | --- | --- |
| 最适合 | 概念讲解、课程图、流程图、视频大纲 | 品牌视觉、landing page、缩略图、动画解释图 |
| 视觉精度 | 低保真、手绘、重点是可讲 | 更接近 Figma / 网页视觉，重点是可交付 |
| 迭代方式 | 手动编辑画布，删字、挪节点、调层级 | AI live update + 截图 steering |
| 输入重点 | 主题、观众、知识来源、图表层级 | 当前 canvas、风格参考、导出尺寸、动画需求 |
| 输出 | 图表、分享链接、课程图、PPT 视觉骨架 | PNG / SVG / 网页视觉稿 / 视频素材 |

判断原则：

- 还在想清楚“怎么讲”时，用 Excalidraw。
- 已经知道要讲什么，想探索“怎么呈现得更像产品 / 品牌 / 页面”时，用 Paper。
- 如果材料只有一篇来源里的工具演示，先并入本页，不单独开工具页。

## 操作指令骨架

### Excalidraw 讲解图

```text
请用 Excalidraw 风格生成一组讲解图。

主题：
[主题]

观众：
[观众]

参考材料：
- YouTube transcript: [链接或文件]
- 我的 Readwise / clipping 摘要: [材料]

要求：
1. 先给出 3-5 张图的结构说明。
2. 每张图只表达一个核心关系。
3. 文字尽量少，优先用框、箭头、分组和层级。
4. 生成结果适合我在视频里全屏讲解。
5. 如果需要查多个来源，请用 sub agents 并行提取上下文。
```

### Paper 视觉画布

```text
请使用 Paper 生成一个可编辑视觉稿。

任务：
[解释图 / 落地页 / 缩略图 / 品牌视觉]

上下文：
[主题、受众、风格参考]

要求：
1. 先看当前 canvas 或参考图。
2. 保留第一屏简洁，不要堆太多文字。
3. 需要动画的部分：[说明]
4. 输出后等待我用截图反馈继续修改。
5. 最终要能导出 PNG / SVG，或转成网页。
```

## 判断输出好不好

- 一眼能看出主线，而不是像文章大纲搬进画布。
- 每张图能支撑一段口播。
- 文本密度低，人工可以在画布上快速改。
- 图与来源材料有关系，不是泛泛模板。
- 可以直接变成 PPT、视频片段或课程讲义。
- 如果是高保真视觉稿，尺寸、层级、留白和导出用途都明确。

## 和剪辑动效的边界

| 任务 | 本页负责 | 剪辑动效页负责 |
| --- | --- | --- |
| 概念讲解图 | 结构、层级、节点、箭头、讲解顺序 | 如果要做成视频 overlay，再决定 timing 和动画 |
| 缩略图 / 封面 / landing page 视觉 | 视觉方向、布局、品牌风格、可编辑稿 | 不负责，除非它要进入视频时间轴 |
| 视频章节图 / 课程图 | 静态视觉骨架和可讲性 | 出现时间、转场、字幕和 render |
| Paper / Excalidraw 画布修改 | 截图 steering、重排、删字、导出 PNG / SVG | 不负责 |
| Remotion / Hyperframes 动效 | 不负责工具细节 | composition、scene、motion beat、preview、render |

## 常见坑

- 默认文字过多，变成“手绘风文档”。
- 一张图塞太多概念，讲解时没有节奏。
- 只追求好看，不服务口播和课程结构。
- 没有给风格参考，生成的图和个人品牌不一致。
- sub agents 并行查资料后没有统一口径，图表逻辑变散。
- 只用抽象语言说“再高级一点”，不截图、不圈问题、不说明导出用途。
- 把 Paper 当普通图片生成器，忽略它可编辑、可迭代、可 live steering 的优势。
- 对工具能力的判断来自单个创作者案例，正式教程前需要验证当前产品状态。
