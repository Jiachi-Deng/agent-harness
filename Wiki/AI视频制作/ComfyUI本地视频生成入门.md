# ComfyUI本地视频生成入门

## 解决什么问题

ComfyUI 适合把 AI 图像/视频生成做成可复用工作流。它的价值不是“输入一句话出完片”，而是把模型、参考图、提示词、采样、放大、视频组合、API 节点连接成一条可检查、可复跑的生产线。

对 AI 视频和短剧来说，ComfyUI 更适合生成：

- 角色参考图、场景图、道具图。
- 图生视频或短镜头素材。
- 可控风格的 B-roll。
- 低分辨率快速迭代后的高分辨率放大版本。
- 与外部闭源模型 API 结合的混合工作流。

本页负责“本地生成工具入口”，不是短剧内容流程页。故事改编、分集、剧本、分镜和连续性管理回到 [[Wiki/AI视频制作/AI短剧与漫剧制作流水线|AI短剧与漫剧制作流水线]]；粗剪、字幕、动效、时间轴和发布回到 [[Wiki/AI视频制作/AI视频剪辑与动态图形工作流|AI视频剪辑与动态图形工作流]]。

## 适合谁

- 愿意本地安装工具、管理模型和节点的人。
- 有一定显卡资源，想降低长期生成成本的人。
- 需要更强控制，而不是只用网页工具一次性生成的人。
- 做短剧、漫剧、游戏资产、产品视觉素材的人。

不适合完全不想处理环境、模型文件、节点报错和显卡限制的人。没有本地 GPU 也能用云 GPU，但需要单独管理成本和文件。

## 硬件与成本判断

从教程材料提炼出的实用判断：

- 8GB VRAM：能跑部分图像模型和经过优化的视频模型，适合入门试验。
- 12-16GB VRAM：更舒服，适合大多数图像和部分视频工作流。
- 24GB VRAM 以上：适合更大模型、更高分辨率和更少妥协。
- 没有独立显卡：可以考虑 RunPod 等云 GPU，但成本从“买硬件”变成“按小时租用”。

这类参数强时效，正式购买前要再查模型和显卡的当前要求。

## 入门安装路径

1. 从 ComfyUI 官网或 GitHub 下载桌面版 / portable 版。
2. 解压到空间充足的位置。
3. 理解几个关键目录：
   - `models/`：模型总目录。
   - `models/checkpoints/`：checkpoint。
   - `models/loras/`：LoRA。
   - `custom_nodes/`：自定义节点。
   - `output/`：生成结果。
4. 先安装 ComfyUI Manager，后续用它安装 custom nodes 和缺失模型。
5. 启动对应 GPU 的运行脚本，打开本地浏览器界面。

## 视频生成最小工作流

教程里推荐新手先用 LTX 2.3 视频模板，因为它相对快、负担较轻，也支持部分音频和对话能力。

最小流程：

1. 打开 template，选择 LTX 视频工作流。
2. 根据缺失提示下载 checkpoint、LoRA、text encoder、upscaler 等模型。
3. 确认每个模型放到正确目录，并在下拉菜单里能选到。
4. 安装 RG3 custom nodes，用进度条和节点高亮观察运行过程。
5. 安装 Video Helper Suite，用 video combine 节点在界面内预览结果。
6. 设置帧率、格式和帧数，例如 24fps、H.264 MP4、短帧数先测试。
7. 写一个短 prompt，先生成低成本版本。
8. 满意后再加放大节点或高质量设置。

## 新手要理解的节点逻辑

不需要一开始理解所有线，但至少要知道：

- prompt / conditioning：告诉模型要往什么方向生成。
- latent：还没变成图像的噪声空间。
- sampler：一步步把噪声变成接近 prompt 的结果。
- VAE decode：把 latent 变成可见像素。
- video combine：把帧组合成视频预览或文件。
- upscaler：把可用但低分辨率的结果放大。

理解这个顺序后，排错会容易很多。哪一个节点变红，就先查它的输入、模型和依赖。

## 短剧/视频制作里的常用组合

- `LTX`：新手视频生成、短镜头测试。
- `Video Helper Suite`：预览和导出视频。
- `Nvidia RTX Video Super Resolution`：视频放大，适合先低分辨率迭代再出高质量。
- `Flux` / 轻量图像模型：角色图、道具图、风格图、局部修改。
- API nodes：把 Google、Runway 或其它闭源模型接进 ComfyUI，而不是每个网页工具各做一遍。
- 参考图 + 风格迁移：用 3D / 低模 / 手绘参考控制构图，再转成目标风格。

## 和短剧流水线的边界

短剧页会定义“应该生成什么”，ComfyUI 页只维护“如何稳定生成”。

| 输入来自短剧页 | ComfyUI 处理 | 输出回到哪里 |
| --- | --- | --- |
| 角色设定、服装、脸部连续性要求 | 生成角色参考图、局部修复、风格统一 | 回到短剧页做角色库和连续性记录 |
| 场景、道具、时代背景 | 生成场景图、道具图、风格图 | 回到短剧页做分镜资产 |
| 单个镜头的景别、动作、首帧 / 尾帧需求 | 生成短镜头、关键帧、图生视频 | 回到剪辑页组合、字幕、配音和发布 |
| 低分辨率草稿 | upscaler、超分、批量复跑 | 回到项目资产库或剪辑时间线 |

如果新材料主要讲节点、模型目录、显存、custom nodes、workflow JSON、采样和放大，进入本页。如果主要讲剧情、镜头语言、分集节奏和人物连续性，进入短剧页。

## 操作指令骨架

```text
目标：生成一个可用于短剧分镜的 3 秒镜头。

画面：
[主体 / 动作 / 场景]

镜头：
[近景 / 中景 / 推镜 / 跟拍 / 轻微手持]

风格：
[写实 / 动画 / 国风 / 赛博朋克 / 微距摄影]

限制：
1. 不要复杂台词，先保证画面可用。
2. 先低分辨率快速迭代。
3. 输出 H.264 MP4。
4. 如果结果可用，再接 upscaler。
```

## 常见坑

- 模型下载对了，但放错目录，导致下拉菜单里找不到。
- custom node 没装或版本不兼容。
- 一上来就高分辨率长视频，显卡爆显存或等待太久。
- 对话生成很容易变成乱码或 gibberish，复杂台词应放到后期配音/字幕。
- prompt 太复杂，导致模型抓不到重点。先生成可控镜头，再逐步增加细节。
- 只保存输出视频，不保存 workflow，后续无法复跑。
