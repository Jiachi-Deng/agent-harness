# AI短剧与漫剧制作流水线

## 解决什么问题

AI 短剧/漫剧不是“写一个故事 prompt 就能出片”。真正可复用的流程要把故事、分集、剧本、审核、角色资产、场景资产、分镜、视频提示词、生成、剪辑和复盘拆开。

这条流水线适合把小说、故事梗概、爆款题材或原创设定，变成一组可生成、可审核、可复跑的视频前置资产。

本页负责“内容生产流程”，不是具体生成工具教程。它维护故事、分集、剧本、连续性、角色资产规格、场景资产规格、分镜和镜头提示词；模型安装、节点排错、显存、upscale 等工具细节回到 [[Wiki/AI视频制作/ComfyUI本地视频生成入门|ComfyUI本地视频生成入门]]；最终剪辑、字幕、配音、动效和发布回到 [[Wiki/AI视频制作/AI视频剪辑与动态图形工作流|AI视频剪辑与动态图形工作流]]。

## 适合谁

- 想把小说或网文改成短剧/漫剧的人。
- 做 AI 短剧账号、短视频连续剧、课程案例的人。
- 需要让多个 agent 分工，而不是一次性写完剧本的人。
- 想建立自己的角色库、场景库、分镜模板和提示词模板的人。

不适合只想生成单张好看的图。单图可以用提示词技巧解决，连续剧需要状态、审核和连续性管理。

## 总流程

```text
原始故事 / 题材
  -> 改编分析
  -> 分集规划
  -> 单集剧本
  -> 业务/合规/连续性审核
  -> 角色/场景/道具资产提示词
  -> 分镜与镜头运动
  -> 首帧/尾帧/参考帧
  -> 视频生成
  -> 剪辑、字幕、配音、发布
  -> 复盘回写 Wiki / 模板
```

## 多 Agent 工作台的分层

`Novel-to-Script-Team` clipping 里最值得保留的不是某个命令，而是把创作生产线拆成了可恢复、可审核、可替换的多层系统：

```text
创作者 / Agent 使用者
  -> 命令：~analyze / ~plan / ~write N / ~review N / ~storyboard-*
  -> Agent：决定谁负责哪个阶段
  -> Skill：定义执行步骤、检查表和产物格式
  -> Reference：提供方法论、标准和合规边界
  -> Scripts：处理检索、生图、反推和视频批处理
  -> Outputs：保存阶段产物、审核报告和项目记忆
```

这个分层适合本库复用到短剧 / 漫剧流程：不要把所有内容塞进一个超级 prompt。故事分析、情绪设计、剧本写作、视觉化审核、连续性记录、分镜、服化道和视频提示词都应该有不同责任。

## 角色分工

| 角色 | 负责什么 | 关键产物 |
| --- | --- | --- |
| `novel-analyzer` | 原文、题材、人物、冲突、可拍性 | 改编分析 |
| `insight-architect` | 隐藏真相、爽点/虐点、观众预期 | 洞察报告 |
| `episode-architect` | 分集目录、节奏、钩子和集间推进 | 分集大纲 |
| `emotion-architect` | 情绪曲线、悬念和认知负荷 | 情绪策略 |
| `script-writer` | 单集剧本、回改和风格吸收 | 剧本文本 |
| `visual-storyteller` | Show Don't Tell，把心理描写改成可拍动作 | 视觉化修订 |
| `review-director` | 业务审核、合规审核和综合复核 | 审核报告 |
| `continuity-recorder` | 人设、称呼、道具、时间线、伏笔 | 连续性记录 |
| `storyboard-director` | 导演分析、分镜策略、审核 | 讲戏本、分镜审核 |
| `storyboard-artist` | 标准分镜和 Seedance / Sora 提示词 | 分镜板、视频 prompt |
| `art-designer` | 人物、场景、道具、服化道、色彩光影 | 资产提示词 |
| `animator` | 镜头运动、动作节奏、转场 | motion prompt |

角色不是为了炫技。只有当某个阶段有独立判断标准、独立产物和独立审核价值时，才值得拆成 agent。

## 与生成工具的分工

短剧流水线的核心资产是“可生成的镜头规格”，不是某个模型或工具。

| 环节 | 本页负责 | 其它页面负责 |
| --- | --- | --- |
| 故事改编 | 题材、冲突、人物关系、世界规则、可拍性 | 不外移 |
| 分集与剧本 | 每集钩子、冲突、悬念、Show Don't Tell、连续性审核 | 不外移 |
| 角色 / 场景 / 道具 | 资产清单、风格、连续性要求、提示词框架 | 具体用 ComfyUI、即梦、Seedance、Sora 等生成时，工具参数和排错回到对应工具页 |
| 分镜 / 镜头 | 景别、机位、动作、首帧 / 尾帧、时长、负面提示词 | 如果需要把分镜做成可讲图或视觉板，进入 [[Wiki/AI视频制作/讲解图与视觉画布工作流|讲解图与视觉画布工作流]] |
| 生成后组合 | 每个镜头验收、失败 prompt 回写、连续性检查 | 时间轴、字幕、配音、转场和 render 进入 [[Wiki/AI视频制作/AI视频剪辑与动态图形工作流|AI视频剪辑与动态图形工作流]] |

判断新材料归位时，先看它解决的是“剧情和镜头设计”还是“工具怎么生成”。前者进本页，后者进工具或剪辑页面。

## 阶段 1：改编分析

目标不是马上写剧本，而是先判断故事能不能拍、怎么拍。

要输出：

- 题材和受众。
- 主线冲突。
- 核心人物关系。
- 世界规则。
- 爽点、虐点、反转点。
- 可拍性问题：心理描写太多、场景太贵、人物太多、设定太复杂。
- 合规和版权风险。

可用 agent 角色：`novel-analyzer`、`insight-architect`。

## 阶段 2：分集规划

短剧不是普通长文缩写。每集都要有推进、钩子和结尾留存。

要输出：

- 分集目录。
- 每集核心事件。
- 每集开头钩子。
- 每集结尾悬念。
- 情绪曲线。
- 信息揭示顺序。
- 跨集伏笔和回收点。

可用 agent 角色：`episode-architect`、`emotion-architect`。

## 阶段 3：剧本写作与审核

单集剧本要从“文字好看”转成“镜头可拍”。

检查重点：

- 是否 Show Don't Tell，把心理活动改成动作、表情、道具、场面。
- 对话是否短、硬、能推动关系。
- 每集是否有明确冲突和转折。
- 人设、称呼、道具、时间线是否连续。
- 是否有平台合规风险。
- 是否参考了有授权的爆款剧本，而不是复制别人的内容。

可用 agent 角色：`script-writer`、`visual-storyteller`、`review-director`、`continuity-recorder`。

## 阶段 4：资产提示词

即梦 AI 教程里有一个实用的图像提示词框架：

```text
风格 + 视角 + 主体 + 背景 + 细节 + 光影 + 质量词
```

例子结构：

- 风格：写实、C4D、国风、赛博朋克、微距摄影、UE5 质感。
- 视角：正面、侧面、斜侧、背面、俯拍、跟拍、微缩摄影。
- 主体：人物、动物、车辆、道具、场景核心。
- 背景：城市夜景、森林、小溪、白底、绿幕、蓝幕。
- 细节：服装、五官、材质、动作、道具、环境元素。
- 光影：清晨雾气、傍晚暖光、夜景霓虹、煤油灯暖光。
- 质量词：电影感构图、胶片颗粒、超精细、高分辨率、最佳品质。

两条练习方法：

1. 在平台“灵感/发现”里找参考图，先分析原提示词结构，再替换主体或视角。
2. 不看原提示词，自己按框架写一遍，生成后再和原提示词对比差距。

## 阶段 5：分镜与视频提示词

分镜要把剧本拆成镜头，而不是把整集剧本塞进视频模型。

每个镜头至少写清：

- 镜头编号。
- 剧情目的。
- 角色和动作。
- 场景与道具。
- 景别和机位。
- 镜头运动。
- 首帧/尾帧需求。
- 视频模型提示词。
- 负面提示词。
- 时长。

标准分镜适合传统影视交付；Seedance / Sora / 图生视频路线更适合直接生成视频提示词和首尾帧需求。

两条路线可以这样分：

| 路线 | 适合什么 | 关键产物 |
| --- | --- | --- |
| 标准分镜流 | 传统影视分镜、图生视频、分镜板交付 | Beat Breakdown、Beat Board、Sequence Board、Motion Prompt |
| Seedance / Sora 流 | 需要直接复制到视频平台的提示词脚本 | 导演分析、人物/场景/道具清单、帧图需求、Seedance 2.0 prompt |

无论走哪条路线，都要加分镜审核：检查模式选择、帧图完整性、提示词可执行性、角色连续性和镜头信息是否过载。

## 状态和完成检查

长篇短剧项目必须能续跑。可复用机制包括：

- 每个剧本目录保存项目状态，例如 `.agent-state.json` 或本库自己的工作日志。
- 同一集内尽量 resume 同一 agent，保留上下文连续性。
- 跨集时重置 agent，避免旧上下文污染和窗口膨胀。
- 最后做 UTF-8 乱码、称呼一致性、文件完整性和关键资产缺漏检查。

这类机制比“提示词写得很长”更重要，因为连续剧最容易坏在称呼漂移、道具消失、时间线不一致和分镜资产漏交。

## 阶段 6：生成、剪辑和复盘

生成环节可以用即梦、Seedance、Sora、ComfyUI、SkyReels 或其它平台。稳定做法是：

1. 先生成角色和场景定稿图。
2. 再生成关键帧。
3. 再用图生视频做短镜头。
4. 每个镜头单独检查，不要一次生成整集。
5. 进入剪辑软件组合，补字幕、配音、音效、转场。
6. 记录失败镜头和有效 prompt，回写模板。

## 关键经验

- 提示词不要越长越好。重点信息放开头和结尾，中间信息要写清楚。
- 多轮改图容易让人物脸部或关键细节漂移。必要时重开会话或重新固定参考图。
- 正写和反写都要有：既写想要什么，也写不要什么。
- 复杂画面先拆成主体、背景、动作、光影，再生成。
- 长项目必须维护 continuity，不然角色称呼、服装、道具和关系会漂。
- 生成只是中段，短剧最终质量仍取决于剧本钩子、节奏、剪辑和声音。
